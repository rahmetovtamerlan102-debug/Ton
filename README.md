# TON Wallet Info Bot (production)

Telegram-бот: по адресу TON-кошелька (Tonkeeper) или `.ton`-домену сразу
показывает баланс/статус в чат, а транзакции, NFT и жетоны — через инлайн-меню.

Источник данных — публичный TonAPI (https://tonapi.io), тот же провайдер,
на котором построен сам Tonkeeper.

## Что показывает

**Сразу в чат при отправке адреса:**
- баланс в TON
- статус аккаунта (active / uninit)
- тип (кошелёк / смарт-контракт), интерфейсы (версия кошелька)
- время последней активности
- пометка, если для перевода нужен memo/tag

**По кнопке «📜 Полный отчёт (транзакции)»:**
- список последних событий кошелька с датой и описанием (переводы, свапы, покупки NFT и т.д.)
- пагинация «Ещё / Назад»
- если событие помечено TonAPI как скам — помечается 🚫

**По кнопке «🖼 NFT»:**
- список NFT на кошельке
- для каждого NFT — статус: выставлен ли он сейчас на продажу/аукцион на маркетплейсе
  (GetGems, Fragment и т.д.) с ценой, или просто лежит на балансе без активного лота
- пометка непроверенных коллекций

**По кнопке «💎 Жетоны»:**
- список Jetton-балансов (токены на TON) с человекочитаемым количеством

## Архитектура

```
ton_wallet_bot/
├── bot.py               # точка входа, сборка Application, логирование
├── config.py             # все настройки из переменных окружения
├── keyboards.py          # инлайн-клавиатуры
├── handlers.py           # команды, обработка сообщений и callback-кнопок
├── services/
│   ├── tonapi.py         # клиент TonAPI: ретраи, кэш, единые исключения
│   └── formatting.py     # форматирование чисел/дат/статусов NFT в HTML
├── requirements.txt
├── Dockerfile
└── .env.example
```

Продакшн-моменты, которые уже учтены:
- **Ретраи с backoff** при 429/5xx и сетевых сбоях (`services/tonapi.py`)
- **TTL-кэш** ответов TonAPI (по умолчанию 20 сек) — гасит всплески повторных
  нажатий "Обновить" и параллельные запросы разных пользователей на один адрес
- **Единые исключения** (`TonApiError`, `AccountNotFound`) вместо разбора кодов
  ответа в каждом хендлере
- **Глобальный error-handler** (`on_error`) — необработанные исключения не роняют бота
- **Конфиг через .env**, ничего не захардкожено
- **Docker-образ** для деплоя

## Установка и запуск

```bash
pip install -r requirements.txt
cp .env.example .env
# впиши BOT_TOKEN (и опционально TONAPI_KEY) в .env
python bot.py
```

## Запуск в Docker

```bash
docker build -t ton-wallet-bot .
docker run -d --name ton-wallet-bot \
  -e BOT_TOKEN="токен_от_BotFather" \
  -e TONAPI_KEY="опционально" \
  --restart unless-stopped \
  ton-wallet-bot
```

## Что дальше можно докрутить

- Перейти с polling на webhook (актуально при высокой нагрузке / деплое за реверс-прокси)
- Вынести кэш в Redis, если бот работает в нескольких инстансах
- Добавить уведомления о новых входящих транзакциях через вебхуки TonAPI
- Добавить persistence сессии пользователя (сейчас `context.user_data` живёт, пока
  жив процесс бота — при рестарте адрес нужно будет прислать заново)

## Про Fragment

Fragment — маркетплейс TON-юзернеймов/номеров/подарков поверх обычной TON-сети,
отдельного API у него нет. Все связанные с ним операции видны в разделе
"Полный отчёт" как обычные ончейн-транзакции кошелька, а листинги NFT на
Fragment (если есть) отображаются в разделе NFT как статус "выставлен на продажу".
